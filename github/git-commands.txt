BASIC GITHUB COMMANDS, CRON JOB SYNTAX AND MAVEN BASICS
(Rough Notes – simple understanding)

---

1. Git / GitHub Basic Commands

Git is a version control system.
It is used to track changes in code and work with teams on the same project.

git init
Used to initialize a new git repository in a project folder.

git clone <repository_url>
Used to copy an existing repository from GitHub to your local machine.

git status
Shows the current state of the repository.
It tells which files are modified, staged, or untracked.

git add filename
Adds a specific file to the staging area.

git add .
Adds all changed files to the staging area.

git commit -m "message"
Saves the staged changes in the repository with a commit message.

git push origin branch_name
Uploads local commits to the remote GitHub repository.

git pull origin branch_name
Downloads the latest changes from the remote repository.

git branch branch_name
Creates a new branch.

git checkout branch_name
Switches to another branch.

git checkout -b branch_name
Creates a new branch and switches to it at the same time.

git merge branch_name
Merges another branch into the current branch.

git log
Displays commit history.

git checkout -- filename
Discards changes in a file and restores it to the last committed state.

Example (real use):
When working on a new feature, developers usually create a new branch, commit changes, push it to GitHub, and then create a pull request.

2. Cron Job Basics

Cron is used in Linux/Unix systems to schedule commands or scripts to run automatically at specific times.

Cron Syntax:

* * * * * command

Fields meaning:

Minute (0–59)
Hour (0–23)
Day of Month (1–31)
Month (1–12)
Day of Week (0–7) → Sunday = 0 or 7

Examples:

* * * * * command
          Runs every minute.

0 * * * * command
Runs every hour.

0 0 * * * command
Runs every day at midnight.

0 6 * * * command
Runs every day at 6 AM.

*/5 * * * * command
Runs every 5 minutes.

0 0 * * 0 command
Runs every Sunday.

30 8 * * 1 command
Runs every Monday at 8:30 AM.

Commands to manage cron jobs:

crontab -e
Opens the cron editor to add or edit cron jobs.

crontab -l
Lists all scheduled cron jobs.

crontab -r
Removes all cron jobs.

Example:
A server can run a backup script every night using a cron job.

3. Maven Basics

Maven is a build automation and dependency management tool used mainly for Java projects.

Common Maven commands:

mvn -version
Checks the installed Maven version.

mvn archetype:generate
Creates a new Maven project.

mvn compile
Compiles the source code.

mvn test
Runs unit tests.

mvn package
Packages the project into a JAR or WAR file.

mvn clean
Removes previously compiled files and build folders.

mvn install
Installs the project into the local Maven repository.

Main Maven Project Structure:

src/main/java
Contains the main application source code.

src/test/java
Contains test cases.

pom.xml
The Project Object Model file.
It manages dependencies, plugins, and build configuration.

Example dependency in pom.xml:

<dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-core</artifactId>
    <version>5.3.0</version>
</dependency>

Common Maven Lifecycle:

clean   → removes old build files
compile → compiles source code
test    → runs tests
package → creates JAR/WAR file
install → installs in local repository
deploy  → publishes the package to a remote repository

---

These tools are commonly used in development:

Git/GitHub → code version control
Cron → scheduling automatic tasks
Maven → building and managing Java projects
